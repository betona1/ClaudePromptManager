PC 연동 Setup

다른 PC에서 Claude Code를 사용하면 이 서버({{ server_url }})에 프롬프트가 자동 기록됩니다.
각 PC마다 아래 설정을 한 번만 해주면 됩니다.

동작 구조
PC (Windows/Linux/Mac)
Claude Code + Hook
→
CPM Server
{{ server_url }}

프롬프트 입력 시 hook이 서버 API로 자동 전송 (5초 타임아웃, 실패해도 Claude Code 동작에 영향 없음)

Claude Code hooks 설정

설정 파일 위치: %USERPROFILE%\.claude\settings.json
(보통 C:\Users\<사용자명>\.claude\settings.json)

settings.json이 없는 경우 (처음 설치)

다운받은 C:\cpm-hooks\settings.json 파일을 복사:

copy C:\cpm-hooks\settings.json %USERPROFILE%\.claude\settings.json

.claude 폴더가 없으면 먼저 만드세요: mkdir %USERPROFILE%\.claude

이미 settings.json이 있는 경우

기존 파일을 메모장으로 열고 "hooks" 부분만 추가 (기존 설정 유지):

"hooks": { "UserPromptSubmit": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python C:\\cpm-hooks\\on_prompt_remote.py" } ] } ], "Stop": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python C:\\cpm-hooks\\on_stop_remote.py" } ] } ] }
기존 JSON에 병합할 때 콤마(,)를 빠뜨리지 마세요!
예: 기존 "permissions": {...} 뒤에 , 추가 후 "hooks" 붙이기

Claude Code hooks 설정

설정 파일 위치: ~/.claude/settings.json

Mode A: 로컬 저장 + 원격 전송

로컬 DB(~/.local/share/cpm/cpm.db)에 저장 + CPM_REMOTE_SERVER로 동시 전송

"hooks": { "UserPromptSubmit": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python3 ~/cpm-hooks/on_prompt.py" } ] } ], "Stop": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python3 ~/cpm-hooks/on_stop.py" } ] } ] }
환경변수 필수: CPM_REMOTE_SERVER={{ server_url }}
이 변수가 없으면 로컬 DB에만 저장됩니다 (현재 이 PC가 이 상태일 수 있습니다).
Mode B: 원격 전용 (로컬 DB 없음)

서버에만 전송. 파일 하나로 동작하므로 가장 간단.

"hooks": { "UserPromptSubmit": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python3 ~/cpm-hooks/remote-only/remote_hook.py prompt" } ] } ], "Stop": [ { "matcher": "", "hooks": [ { "type": "command", "command": "python3 ~/cpm-hooks/remote-only/remote_hook.py stop" } ] } ] }
settings.json이 없는 경우
# Mode A (로컬+원격) cp ~/cpm-hooks/settings-local-remote.json ~/.claude/settings.json # Mode B (원격 전용) cp ~/cpm-hooks/settings-remote-only.json ~/.claude/settings.json
이미 settings.json이 있는 경우

기존 파일을 열고 위의 "hooks" 부분만 병합하세요.

설정 완료! 이제 이 PC의 모든 Claude Code 프롬프트가 자동 기록됩니다.
Linux/Mac 문제 해결
  • python3 설치 확인: python3 --version
  • 환경변수 확인: echo $CPM_REMOTE_SERVER — 값이 비어있으면 셸 재시작 필요
  • 서버 연결 확인: curl -s {{ server_url }}/api/ | head -1

기존 기록 가져오기

tmux — 핸드폰에서 원격 작업하기

tmux를 사용하면 서버에 터미널 세션을 만들어 놓고, 핸드폰(SSH)이나 다른 PC에서 같은 화면을 실시간 공유할 수 있습니다.
연결이 끊어져도 세션은 살아있어서 나중에 다시 이어서 작업 가능합니다.

tmux 세션 공유
PC (서버)
tmux new -s work
←
핸드폰 / 다른 PC
SSH → tmux attach

양쪽에서 같은 화면을 실시간으로 보고, 한쪽에서 타이핑하면 다른 쪽에도 보입니다

0

설치

sudo apt install -y tmux

Mac: brew install tmux  |  이미 설치되어 있으면 건너뛰세요.

기본 사용법
1

세션 만들기

서버(PC)에서 새 tmux 세션을 시작합니다:

tmux new -s work

work은 세션 이름입니다. 원하는 이름으로 바꿔도 됩니다.

2

핸드폰에서 붙기

핸드폰(cpmSSH 등)에서 같은 서버에 SSH 접속 후:

tmux attach -t work

이제 PC와 핸드폰이 같은 화면을 보게 됩니다!

3

세션에서 나가기 (세션은 유지)

Ctrl+B → D

Ctrl+B를 먼저 누르고, 손을 뗀 후 D를 누릅니다.
세션은 백그라운드에서 계속 실행됩니다. 나중에 tmux attach -t work로 다시 붙으면 됩니다.

명령어

실전 시나리오
핸드폰에서 Claude Code 사용하기
# PC에서 (한 번만) tmux new -s work claude # Claude Code 실행 # 핸드폰에서 SSH 접속 후 tmux attach -t work # 같은 화면 공유! # Claude Code를 핸드폰에서 사용 가능
화면 분할로 멀티태스킹
# 세션 안에서 Ctrl+B → % # 좌우 분할 # 왼쪽: Claude Code, 오른쪽: 파일 편집 Ctrl+B → 방향키 # 패널 이동
퇴근 후 이어서 작업
# 퇴근 전 — 세션 놔두고 나가기 Ctrl+B → D # 다음날 (또는 핸드폰에서) tmux attach -t work # 어제 상태 그대로!
CPM + tmux 조합의 장점: 서버에 CPM hooks가 설치되어 있으면, tmux 안에서 Claude Code를 실행할 때 핸드폰에서 타이핑해도 프롬프트가 자동 기록됩니다. 실제 실행은 서버에서 일어나므로 hooks가 정상 동작합니다.
주의사항
  • tmux는 같은 서버 안에서만 세션 공유 가능 (80번 세션을 200번에서 attach 불가)
  • 화면 크기가 다르면 작은 쪽에 맞춰짐 → 핸드폰 가로 모드 추천
  • 세션 안에서 exit 입력하면 세션이 완전 종료됨 (나가기만 하려면 Ctrl+B → D)
  • 스크롤이 안 되면 Ctrl+B → [로 스크롤 모드 진입
