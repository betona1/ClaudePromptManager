■ STEP 0. 크롤링 이란? (왜 배워야 하나?)
◎ 크롤링 = 웹사이트에서 정보를 자동으로 수집하는 기술
◎ 쇼핑몰 사장님이 크롤링으로 할 수 있는 것들 :
경쟁사 가격 자동 감시매일 새벽에 경쟁사 10곳 가격을 엑셀로 정리 → 내 가격 전략
트렌드 상품 자동 발굴쿠팡 · 네이버쇼핑 베스트셀러 매일 추적 → 신상품 아이디어
리뷰 자동 수집 분석내 상품 + 경쟁사 리뷰 모아 AI 에게 분석 요청 → 개선점
공급처 재고 자동 확인도매처 재고 · 가격 주기적으로 확인 → 품절 전 발주
키워드 · 해시태그 추적인스타 · 블로그 인기 키워드 수집 → SEO · 광고 전략
이미지 · 상세페이지 수집경쟁사 상세페이지 구성 분석 → 디자인 벤치마킹
■ STEP 1. 크롤링 vs 스크래핑 — 정확히 구분하기
크롤링 (Crawling)링크를 따라가며 여러 페이지를 돌아다니는 것 (검색엔진 방식)
스크래핑 (Scraping)특정 페이지에서 원하는 정보만 긁어오는 것
실무에서는대부분 섞어 씀 → "크롤링" 이라고 통칭
■ STEP 2. 사이트 종류 파악 — 내 상대 먼저 알기
정적 사이트 (Static)서버가 HTML 통째로 주는 사이트 → 쉬움 (블로그, 뉴스)
동적 사이트 (Dynamic)JavaScript 로 내용 채우는 사이트 → 어려움 (쿠팡, 네이버)
SPA한 페이지 안에서 계속 바뀌는 사이트 (React, Vue 사이트)
API 있는 사이트숨겨진 API 찾으면 개꿀! (제일 깔끔한 방법)
■ STEP 3. 크롤링 도구 4대 천왕 — 언제 뭘 쓸까?
도구속도난이도언제 쓰나
Requests + BS4★★★★★쉬움정적 사이트 (블로그, 뉴스)
Selenium★★보통로그인 필요, 복잡한 조작
Playwright★★★★보통요즘 사이트 대부분 (쿠팡 등)
Scrapy★★★★★어려움수천~수만 페이지 대규모
◎ 쇼핑몰 사장님 추천 공식:
1. 일단 Requests + BS4 로 시도 → 70%는 이걸로 됨
2. 안 되면 Playwright 로 (진짜 브라우저 열어서)
3. Claude 에게 "이 사이트 어떤 도구 써야 해?" 물어보면 답해줌
■ STEP 4. HTML 구조 이해 — 크롤링의 심장
HTML웹페이지의 뼈대 코드 (F12 눌러서 볼 수 있음)
태그 (Tag)<div>, <a>, <img> 같은 것 — 요소의 종류
클래스 (class)태그에 붙은 이름표 (class="price")
ID태그에 붙은 고유 번호표 (id="product-1")
CSS 선택자원하는 요소 콕 집는 법 (.price, #product-1)
XPathCSS 선택자보다 정교한 주소 (복잡한 구조용)
DOM브라우저가 해석한 HTML 구조 트리
개발자도구 (F12)웹페이지 구조 분석하는 필수 도구 → Claude 에 스샷 붙이기
■ STEP 5. HTTP 통신 용어 — 서버와 대화하는 법
GET 요청"이 페이지 주세요" 요청 (조회용)
POST 요청"이 정보 보내요" 요청 (로그인, 검색)
헤더 (Headers)요청할 때 붙이는 꼬리표 (User-Agent, 쿠키 등)
상태 코드200 = 성공 / 404 = 없음 / 403 = 차단 / 500 = 서버오류
응답 (Response)서버가 돌려주는 결과 (HTML, JSON)
세션 / 쿠키로그인 유지용 → 크롤링할 때 중요
■ STEP 6. 왜 차단 당할까? — 사이트가 막는 방법
◎ 요즘 사이트는 5단계로 봇을 잡습니다 :
1단계 : User-Agent 체크브라우저가 아니면 차단 (파이썬 기본값은 Python-urllib)
2단계 : IP 주소 차단같은 IP에서 너무 많이 요청하면 Rate Limit 발동
3단계 : JavaScript 실행봇은 JS 못 돌림 → 내용 안 보임
4단계 : 행동 패턴 분석마우스 안 움직이고 너무 빨리 클릭 = 봇
5단계 : CAPTCHA"나는 로봇이 아님" 체크, 이미지 맞추기
■ STEP 7. 차단 우회 기술 — 합법 범위 내에서
User-Agent 위장크롬 브라우저인 척 꼬리표 달기 (기본 중 기본)
Referer 헤더"네이버에서 왔어요" 거짓말 (사이트별 요구사항)
딜레이 (time.sleep)요청 사이에 2~5초 간격 두기 → 가장 중요!
랜덤 딜레이1~7초 랜덤 — 사람처럼 보이게
프록시 (Proxy)IP 여러 개 돌려쓰기 (유료 서비스 많음)
VPN내 IP 완전히 바꾸기 (개인 크롤링용)
쿠키 · 세션 유지로그인 상태 기억시켜서 요청
헤드리스 브라우저Playwright 로 진짜 크롬 띄워서 우회
Stealth 플러그인봇 티 안 나게 브라우저 속이기
CAPTCHA 서비스2Captcha, CapSolver — 유료로 풀어줌 (최후의 수단)

▶ 쇼핑몰 실전 사례 5가지
■ 사례 1. 경쟁사 가격 매일 추적하기

tmux — 핸드폰에서 원격 작업하기

tmux를 사용하면 서버에 터미널 세션을 만들어 놓고, 핸드폰(SSH)이나 다른 PC에서 같은 화면을 실시간 공유할 수 있습니다.
연결이 끊어져도 세션은 살아있어서 나중에 다시 이어서 작업 가능합니다.

tmux 세션 공유
PC (서버)
tmux new -s work
←
핸드폰 / 다른 PC
SSH → tmux attach

양쪽에서 같은 화면을 실시간으로 보고, 한쪽에서 타이핑하면 다른 쪽에도 보입니다

0

설치

sudo apt install -y tmux

Mac: brew install tmux  |  이미 설치되어 있으면 건너뛰세요.

기본 사용법
1

세션 만들기

서버(PC)에서 새 tmux 세션을 시작합니다:

tmux new -s work

work은 세션 이름입니다. 원하는 이름으로 바꿔도 됩니다.

2

핸드폰에서 붙기

핸드폰(cpmSSH 등)에서 같은 서버에 SSH 접속 후:

tmux attach -t work

이제 PC와 핸드폰이 같은 화면을 보게 됩니다!

3

세션에서 나가기 (세션은 유지)

Ctrl+B → D

Ctrl+B를 먼저 누르고, 손을 뗀 후 D를 누릅니다.
세션은 백그라운드에서 계속 실행됩니다. 나중에 tmux attach -t work로 다시 붙으면 됩니다.

# 퇴근 전 — 세션 놔두고 나가기 Ctrl+B → D # 다음날 (또는 핸드폰에서) tmux attach -t work # 어제 상태 그대로!
CPM + tmux 조합의 장점: 서버에 CPM hooks가 설치되어 있으면, tmux 안에서 Claude Code를 실행할 때 핸드폰에서 타이핑해도 프롬프트가 자동 기록됩니다. 실제 실행은 서버에서 일어나므로 hooks가 정상 동작합니다.
주의사항
  • tmux는 같은 서버 안에서만 세션 공유 가능 (80번 세션을 200번에서 attach 불가)
  • 화면 크기가 다르면 작은 쪽에 맞춰짐 → 핸드폰 가로 모드 추천
  • 세션 안에서 exit 입력하면 세션이 완전 종료됨 (나가기만 하려면 Ctrl+B → D)
  • 스크롤이 안 되면 Ctrl+B → [로 스크롤 모드 진입

Telegram Bot 만들기 (완전 초보용)

Part A. 봇 만들기 (2분)
1

BotFather 열기

아래 링크를 폰에서 누르세요. Telegram 앱이 바로 열립니다.

BotFather 열기 (폰에서 클릭)
PC에서 보고 있다면? 폰 Telegram 앱 열고 → 상단 검색 → BotFather 입력 → 파란 체크표시 있는 계정 선택
확인: BotFather와 채팅방이 열렸나요? → 하단에 START 또는 메시지 입력창이 보이면 OK
2

/newbot 보내기

채팅창에 아래를 그대로 입력해서 보내세요:

/newbot
확인: BotFather가 "Alright, a new bot. How are we going to call it?" 라고 답했나요? → 다음으로
3

봇 이름 입력 (2번 대답해야 함!)

여기가 제일 헷갈리는 부분! BotFather가 두 번 물어봅니다. 하나는 "이름", 하나는 "ID" 입니다.

첫 번째 질문 — "이름을 뭘로 할래?" (표시되는 이름)

Alright, a new bot. How are we going to call it? Please choose a name for your bot.

Part B. Chat ID 알아내기 (1분)
5

Chat ID란?

봇이 메시지를 보낼 대상(나)의 고유번호입니다.

username(@myname)이 아니고, 순수 숫자입니다 (예: 123456789)

가장 쉬운 방법

아래 링크를 눌러 @userinfobot을 여세요:

userinfobot 열기 (폰에서 클릭)

열리면 /start 또는 아무 메시지를 보내세요:

/start
@joacham
Id: 5765213489
First: Joacham
Lang: ko

Id: 뒤의 숫자를 복사하세요. 그게 Chat ID입니다.

확인: 숫자만 복사했나요? (공백, @ 없이 숫자만!) → 다음으로

봇에게 먼저 /start 보내기

이걸 안 하면 Test가 절대 안 됩니다!
Telegram 봇은 사용자가 먼저 말을 걸어야만 그 사용자에게 메시지를 보낼 수 있습니다.

Part D. CPM에 등록 (30초)
7

토큰 + Chat ID 입력

이 페이지 위의 Telegram 봇 연동 → [+] 버튼 클릭:

  • Bot Token: 4단계에서 복사한 토큰 (예: 7123456789:AAH_abc...)
  • Chat ID: 5단계에서 복사한 숫자 (예: 5765213489)

[추가] 클릭 → 토큰 자동 검증 → 봇 이름이 자동으로 채워짐

[Test] 버튼 → Telegram에 메시지가 오면 성공!

성공! Telegram에 "CPM Test Message"가 왔으면 설정 완료!
